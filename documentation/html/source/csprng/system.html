csprng.system

ASN.1 Codecs, as specified in ITU X.690, including BER, DER, and PER

A Cryptographically-Secure Pseudo-Random Number Generator that uses system APIs as its source of secure random bytes. In the case of Windows, it uses the BCryptGenRandom, CryptGenRandom, and RtlGenRandom libraries in that order of fallback. On POSIX-ish systems, it uses the pseudo-device, /dev/random, as its source of secure random bytes.

    • isUsingCryptographyNextGenerationApplicationProgrammingInterface
      nothrow @nogc @property @safe bool isUsingCryptographyNextGenerationApplicationProgrammingInterface();
      Returns boolean indicating whether this library is using the Windows Cryptography: Next Generation API to generate random bytes from the BCryptGenRandom API function.
      More specifically, this library returns true if Bcrypt.dll was found, loaded, and all three requisite functions could be loaded from it, which are:
      • BCryptOpenAlgorithmProvider
      • BCryptCloseAlgorithmProvider
      • BCryptGenRandom

    • isUsingCryptoApplicationProgrammingInterface
      nothrow @nogc @property @safe bool isUsingCryptoApplicationProgrammingInterface();
      Returns boolean indicating whether this library is using the Windows Crypto API to generate random bytes from the CryptGenRandom API function.
      More specifically, this library returns true if advapi32.dll was found, loaded, and all three requisite functions could be loaded from it, which are:
      • CryptAcquireContext
      • CryptReleaseContext
      • CryptGenRandom
    • isUsingRtlGenRandom
      nothrow @nogc @property @safe bool isUsingRtlGenRandom();
      Returns a boolean indicating whether this library was able to load advapi32.dll and RtlGenRandom from it, and will use RtlGenRandom to obtain secure random bytes.
    • getBytes
      @system void[] getBytes(in size_t length);
      Returns the specified number of cryptographically-secure pseudo-random bytes, using one of the system APIs.

      Throws

      • CSPRNGException if one of the functions from the automatically-select cryptography API could not be loaded.

    • this
      @system this();
      The constructor for a CSPRNG. When a CSPRNG is created with this constructor, the relevant libraries and functions from them are loaded. If they cannot be loaded, the constructor throws a CSPRNGException and the CSPRNG will not be created.
      This constructor first attempts to load the Windows

      Cryptography: Next Generation API. If that cannot be loaded, it attempts to load the CryptoAPI. If that cannot be loaded, it attempts to load the RtlGenRandom function. If all of those fail, the CSPRNGException is thrown, since no source of cryptographically-secure pseudo-random bytes can be accessed.
